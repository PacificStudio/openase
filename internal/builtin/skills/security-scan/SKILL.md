# Security Scan

At minimum, cover:

- Authentication and authorization boundaries
- Command / SQL / template injection risks
- Sensitive data exposure
- Dependency versions and known vulnerabilities
- Whether default configuration is too permissive

When you find problems, provide a reproducible path and a concrete fix recommendation.
