# 结构化可观测性清单

OpenASE 在边界输入和模糊的运行时状态转换处产生结构化日志。本清单是新代码和可观测性审计的最低契约。

## 核心字段

在边界处可用时，包含稳定的关联字段：

- `component`
- `operation`
- `error`
- `request_id`
- `trace_id`
- `span_id`
- `project_id`
- `org_id`
- `ticket_id`
- `workflow_id`
- `run_id`
- `agent_id`
- `provider_id`
- `machine_id`
- `session_id`
- `transport_mode`
- `workspace_root`
- `failure_stage`

## 边界规则

- 在原始外部或不稳定输入进入系统的边界处记录一次日志。
- 优先使用结构化字段/值对，而非自由格式的消息文本。
- 捕获标识符、负载结构、键计数或配置键，而非转储大型或敏感负载。
- 解析成功后，不要在下游业务逻辑中分散重复的验证日志。
- 对畸形或降级的边界输入使用 `warn`，对服务端故障或不可能的状态使用 `error`。

## 必须审查的区域

- HTTP/API 解析：请求路径、路由、请求 ID、状态码、错误码、路径参数、查询键、内容长度。
- CLI/配置加载：配置文件路径、覆盖键、失败的字段或解析器、已知时的服务器模式。
- SSE 和入站事件解码：主题、事件类型、负载字节数、发布时间、范围 ID。
- 工作流和工单钩子：钩子名称、范围、策略、命令、持续时间、结果、阻塞状态、相关 ID。
- 调度器/重试/协调：跳过原因、容量快照、活跃计数、重试退避、暂停原因、释放的运行时 ID。
- 上游 Provider/API 调用：方法、端点或操作、状态码、上游请求 ID、相关项目/Provider ID。
- 通知投递：规则 ID、通道 ID/类型、配置键、事件类型、分发结果。
- 历史/部分 DB 映射：当持久化数据无法解析为预期的领域形式时记录日志。

## 解析，而非验证

- 将不稳定输入一次性解析为领域类型。
- 在解析边界处记录解析失败。
- 让业务服务在已解析的领域类型上操作，而非重复防御性检查。

## 噪声控制

- 不要为每个正常路径的辅助调用添加 info 日志。
- 优先使用边界日志、状态转换日志、警告和失败摘要。
- 当跳过或失败可能反复发生时，记录关控原因和容量快照，而非原始负载转储。
