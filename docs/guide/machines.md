# Machines（执行机器）

## 这是什么？

Machine 是 Agent 的**运行环境**。Agent 本身是"大脑"，Machine 是"手脚"——Agent 需要在某台机器上执行代码、运行命令。可以是你的本地开发机，也可以是远程服务器或云端虚拟机。

## 基本概念

| 概念 | 说明 |
|------|------|
| **机器（Machine）** | 一个已注册的执行环境，包含连接信息（主机、端口、用户名） |
| **健康状态** | 机器当前的连接和资源状况（在线 / 离线 / 异常） |
| **资源指标** | CPU、内存、磁盘使用情况的快照 |
| **连接探测（Probe）** | 测试机器连通性并收集诊断信息 |

## 常用操作

### 添加机器

1. 进入 Machines 页面
2. 点击"添加机器"
3. 填写以下信息：
   - **名称**：用于在平台内标识的名字
   - **主机地址**：IP 或域名
   - **SSH 端口**：默认 22
   - **SSH 用户名**：用于登录的用户
4. 点击"测试连接"验证

### 监控机器

- 查看每台机器的健康状态（在线 / 离线 / 异常）
- 查看资源使用情况（CPU、内存、磁盘）
- 点击"刷新"获取最新状态
- 支持实时流式健康更新（SSE）

### 删除机器

确认没有 Agent 正在使用该机器后，可以删除。

## 小贴士

- 建议至少注册一台机器后再创建工单，否则 Agent 无处执行
- 如果 Agent 执行失败，优先检查目标机器的连接状态和可用资源
- 定期关注机器的资源指标，避免因磁盘满或内存不足导致执行失败
