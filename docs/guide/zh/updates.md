# Updates（项目动态）

## 这是什么？

Updates 是**人工发布的项目进展**。与自动生成的 [Activity](./activity.md) 日志不同，Updates 是团队成员主动发布的状态更新、里程碑通报或讨论帖。你可以把它理解为项目内部的"公告板"或"周报"。

## 基本概念

| 概念 | 说明 |
|------|------|
| **动态帖（Thread）** | 一条项目进展记录，支持 Markdown |
| **评论（Comment）** | 帖子下方的讨论回复 |
| **修订历史** | 帖子和评论的编辑记录，方便审计追溯 |

## 常用操作

### 发布动态

1. 进入 Updates 页面
2. 点击"新建动态"
3. 填写标题和内容（支持 Markdown）
4. 发布

### 参与讨论

- 在动态帖下方添加评论
- 编辑或删除自己的评论
- 查看修订历史了解改动

## Updates vs Activity

| | Updates | Activity |
|---|---------|----------|
| **谁产生** | 团队成员手动发布 | 系统自动生成 |
| **内容** | 主观的进展总结、决策、讨论 | 客观的事件记录 |
| **可编辑** | 是（支持修订历史） | 否（只读） |
| **用途** | 沟通和记录"为什么" | 追踪和审计"做了什么" |

## 使用场景

- 每日/每周项目进展总结
- 重要决策记录（如"我们决定用方案 A 而不是方案 B"）
- Agent 完成关键任务后的人工验收报告
- 阶段性里程碑通报

## 小贴士

- Updates 适合记录"为什么这么做"，Activity 适合查看"做了什么"
- 善用 Markdown 格式让动态更易读
- 重要决策建议发布到 Updates，避免口头讨论后遗忘
