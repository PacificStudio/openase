# Tickets（工单）

## 这是什么？

工单是整个平台的**核心单元**。每一个需要完成的任务——无论是修 Bug、写新功能、还是跑测试——都以工单的形式存在。你可以把它理解为 Jira 的 Issue 或 GitHub 的 Issue，但它不仅仅是记录，更是 AI Agent 的"任务指令"。

## 基本概念

| 概念 | 说明 |
|------|------|
| **状态（Status）** | 工单当前所处阶段，如"待处理"、"进行中"、"已完成"。状态在 [Settings](./settings.md) 中自定义 |
| **工作流（Workflow）** | 工单关联的执行模板，决定了 Agent 如何处理这个工单。详见 [Workflows](./workflows.md) |
| **优先级（Priority）** | 工单的紧急程度 |
| **依赖关系** | 工单之间可以设置"阻塞"和"被阻塞"关系 |
| **父子关系** | 大工单可以拆分为多个子工单 |

## 视图模式

- **看板视图（Board）**：按状态列展示工单，拖拽即可切换状态
- **列表视图（List）**：表格形式展示所有工单，适合批量查看

## 常用操作

### 创建工单

1. 点击"新建工单"
2. 填写标题、描述（支持 Markdown）
3. 选择工作流和初始状态
4. （可选）设置优先级、依赖关系、仓库范围

### 管理工单

- **评论**：在工单下方添加评论进行讨论，支持查看编辑历史
- **外部链接**：关联 GitHub PR、设计文档等外部资源
- **仓库范围**：限定 Agent 只能访问特定文件或目录
- **归档**：完成或不再需要的工单可以归档（可在 [Settings](./settings.md) 中恢复）

### 筛选工单

- 按工作流类型筛选
- 按分配的 Agent 筛选
- 按状态筛选

## 小贴士

- 工单描述越清晰，Agent 执行效果越好。建议包含：需求背景、预期结果、相关代码位置
- 利用父子工单将复杂任务拆解为可独立执行的小任务
- 设置仓库范围可以限制 Agent 的访问范围，提高安全性和执行效率
