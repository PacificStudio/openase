# Scheduled Jobs（定时任务）

## 这是什么？

定时任务让你可以**按计划自动创建工单**。比如每天凌晨自动创建一个"运行回归测试"的工单，或者每周一自动创建"代码审查"工单。它就像一个内置的 Cron 调度器，让重复性工作完全自动化。

## 基本概念

| 概念 | 说明 |
|------|------|
| **Cron 表达式** | 标准的 5 字段格式：`分 时 日 月 周` |
| **工单模板** | 预填的工单数据（标题、描述、工作流绑定） |
| **执行记录** | 上次运行时间和下次运行时间 |

## Cron 表达式速查

Cron 表达式由 5 个字段组成，用空格分隔：

```
┌───────────── 分钟 (0-59)
│ ┌───────────── 小时 (0-23)
│ │ ┌───────────── 日 (1-31)
│ │ │ ┌───────────── 月 (1-12)
│ │ │ │ ┌───────────── 星期 (0-6, 0=周日)
│ │ │ │ │
* * * * *
```

**常用示例**：

| 表达式 | 含义 |
|--------|------|
| `0 8 * * *` | 每天早上 8:00 |
| `0 8 * * 1` | 每周一早上 8:00 |
| `0 0 1 * *` | 每月 1 号零点 |
| `*/30 * * * *` | 每 30 分钟 |
| `0 9-17 * * 1-5` | 工作日每小时（9:00-17:00） |

## 常用操作

### 创建定时任务

1. 进入 Scheduled Jobs 页面
2. 点击"新建定时任务"
3. 填写名称和 Cron 表达式
4. 配置工单模板：
   - **标题**：自动创建的工单标题
   - **描述**：工单描述内容
   - **工作流**：关联的 [Workflow](./workflows.md)
5. 保存并启用

### 管理定时任务

| 操作 | 说明 |
|------|------|
| **启用 / 禁用** | 临时关闭而不删除 |
| **手动触发** | 不等到下次调度时间，立即执行一次 |
| **编辑** | 修改 Cron 表达式或工单模板 |
| **删除** | 永久移除定时任务 |

## 使用场景

| 场景 | Cron 表达式 | 说明 |
|------|-------------|------|
| **每日回归测试** | `0 2 * * *` | 每天凌晨 2 点创建测试工单 |
| **每周代码审查** | `0 9 * * 1` | 每周一上午 9 点创建 review 工单 |
| **依赖安全检查** | `0 10 * * 3` | 每周三上午 10 点检查依赖漏洞 |
| **文档同步** | `0 0 1 * *` | 每月 1 号更新文档 |

## 小贴士

- 确保定时任务绑定的工作流已经配置好 Agent 和状态绑定，否则自动创建的工单不会被执行
- 合理设置执行频率，避免产生大量未处理的工单
- 使用"手动触发"功能来测试定时任务是否按预期工作
- 关注"下次运行时间"确认 Cron 表达式是否正确
