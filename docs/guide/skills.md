# Skills（技能）

## 这是什么？

Skills 是**可复用的自动化行为包**。如果说 [Workflow](./workflows.md) 是"岗位职责"，技能就是"专业技能证书"。它们包含脚本、代码片段或操作规范，可以被绑定到多个工作流中，让不同的工作流共享相同的能力。

## 基本概念

| 概念 | 说明 |
|------|------|
| **内置技能（Built-in）** | 平台自带的技能，如 Git 操作、包管理等 |
| **自定义技能（Custom）** | 用户为项目创建的技能 |
| **技能包（Bundle）** | 一个技能包含的所有文件集合 |
| **技能绑定** | 将技能附加到工作流，Agent 执行时可以调用 |

## 技能包结构

一个技能由以下类型的文件组成：

| 文件类型 | 说明 |
|----------|------|
| `entrypoint` | 入口文件，Agent 执行时的起点 |
| `metadata` | 元数据描述文件 |
| `script` | 可执行脚本 |
| `reference` | 参考文档 |
| `asset` | 静态资源 |

## 常用操作

### 浏览技能

进入 Skills 页面查看所有可用技能。支持筛选：

- **全部**：所有技能
- **内置**：平台自带的技能
- **自定义**：项目中创建的技能
- **已禁用**：被暂时关闭的技能

### 创建自定义技能

1. 点击"新建技能"
2. 填写名称和描述
3. 编写技能内容（Markdown 模板）
4. 保存

### 管理技能

- **启用 / 禁用**：临时关闭技能而不删除
- **编辑**：修改技能内容和元数据
- **查看绑定**：了解哪些工作流正在使用这个技能
- **删除**：移除不再需要的自定义技能（内置技能不可删除）

## 使用场景

| 场景 | 技能示例 |
|------|----------|
| **Git 操作规范** | 创建分支、提交代码、发起 PR 的标准流程 |
| **代码规范检查** | 绑定 lint/format 脚本 |
| **测试执行** | 标准化的测试运行和报告生成 |
| **部署流程** | 自动化部署步骤 |
| **文档生成** | 自动生成 API 文档或 Changelog |

## 小贴士

- 优先使用内置技能，它们经过充分测试
- 自定义技能适合封装项目特有的流程和规范
- 一个技能可以绑定到多个工作流，修改后所有绑定的工作流都会生效
- 编写技能时，注意入口文件的可执行权限设置
