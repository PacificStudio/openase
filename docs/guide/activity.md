# Activity（活动日志）

## 这是什么？

Activity 是**系统自动生成的事件流**。项目中发生的每一件事——工单创建、Agent 开始执行、状态变更、执行完成——都会被记录在这里。它就像项目的"监控大屏"，让你对所有活动一目了然。

## 基本概念

| 概念 | 说明 |
|------|------|
| **事件（Event）** | 一条活动记录，包含时间戳、事件类型和详情 |
| **事件类型** | 工单事件、Agent 事件、工作流事件、机器事件、用户操作等 |
| **事件元数据** | 结构化的上下文信息（如涉及的工单 ID、Agent 名称等） |

## 事件类型

| 类型 | 示例 |
|------|------|
| **工单事件** | 工单创建、状态变更、评论添加、归档 |
| **Agent 事件** | Agent 领取工单、开始执行、执行完成、执行失败 |
| **工作流事件** | 工作流创建、更新、退役 |
| **机器事件** | 机器上线、离线、健康检查失败 |
| **用户操作** | 手动操作记录 |

## 常用操作

### 查看活动

- 进入 Activity 页面即可看到按时间排列的事件流（最新的在最前）
- 支持实时更新（SSE 推送），无需手动刷新

### 筛选事件

- **按类型筛选**：全部 / 工单 / Agent / 工作流等
- **按关键词搜索**：工单标识、Agent 名称、事件内容

## 使用场景

- **排查问题**：Agent 执行失败时，在 Activity 中查看完整事件链
- **了解生命周期**：追踪某个工单从创建到完成的每一步
- **监控活跃度**：了解项目整体运行状况
- **回溯历史**：查看某个时间段内发生了什么

## 小贴士

- Activity 是只读的，你无法手动添加或删除事件
- 如果需要发布主观的项目进展，请使用 [Updates](./updates.md)
- Activity 中的事件包含元数据链接，点击可以跳转到相关的工单或 Agent
